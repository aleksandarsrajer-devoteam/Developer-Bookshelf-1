# 📚 Developer Bookshelf 1 — Cloud Architecture & GitOps Pipeline

Welcome to **Developer Bookshelf 1**! This is a modern, containerized Node.js application organized as a **Monorepo**, where both the application source code and the complete Infrastructure as Code (IaC) live under the same roof.

The entire ecosystem runs on **Google Cloud Platform (GCP)**, featuring a fully automated, hands-off **GitOps deployment pipeline**.

---

## 🏗️ Infrastructure Overview

The underlying cloud infrastructure is fully provisioned and managed via **Terraform** (located in the `/infra` directory). It consists of the following cloud-native components:

* **Google Cloud Run:** Fully managed serverless platform hosting our Node.js container.
* **Google Firestore (Native Mode):** A scalable NoSQL document database used for storing book metadata.
* **Google Cloud Storage (Bucket):** Public, high-availability object storage used for hosting book cover images.
* **Artifact Registry:** A secure private Docker repository for storing and versioning compiled container images.
* **Custom IAM Roles & Service Accounts:** Adhering to the *Principle of Least Privilege*. The application runs under a dedicated `bookshelf-run-sa` account with scoped access to Firestore and Storage, while the CI/CD pipeline utilizes strict custom roles.

## ⚡ Deep Dive: How Cloud Build & Cloud Deploy Work Together

This project strictly follows the **GitOps philosophy**. No manual infrastructure or code updates are done via the GCP Console. The exact mechanics of how a single `git push` transforms into a live website involve a highly coordinated handoff between **Cloud Build** and **Cloud Deploy**.

[ GitHub Push ] ──> [ Cloud Build (CI) ] ──> [ Cloud Deploy (CD) ] ──> [ Cloud Run ]

### 🧪 1. Continuous Integration (CI) — Google Cloud Build
Cloud Build acts as our automated compilation factory. It is triggered instantly on every push to the `main` branch via a webhook connection to GitHub. Guided by `cloudbuild.yaml`, it executes the following atomic operations:

1.  **Code Ingestion:** Cloud Build spins up an ephemeral worker, pulls the latest code commit, and caches the workspace.
2.  **Containerization (Docker Build):** It executes the `Dockerfile`, installing dependencies and compiling the Node.js environment. To ensure strict traceability, every image is tagged with the unique **Git Short SHA** (`:$SHORT_SHA`) instead of mutable tags like `latest`.
3.  **Registry Push:** The freshly baked image is uploaded directly into the *Artifact Registry* (`bookshelf-repo`).
4.  **The Hand-off:** Cloud Build triggers Cloud Deploy by executing the `gcloud deploy releases create` command. It packages the `skaffold.yaml` and `service.yaml` manifests from the root directory and sends them to Cloud Deploy along with the exact URL of the new Docker image.

### 🚚 2. Continuous Delivery (CD) — Google Cloud Deploy
Cloud Deploy is the orchestrator responsible for safe, declarative, and structured rollouts. It solves the classic conflict where Terraform and application deployment overlap by isolating infrastructure states from application releases.

1.  **Manifest Parsing via Skaffold:** Cloud Deploy initializes **Skaffold** in the background. Skaffold acts as the runtime manifest engine. It reads `skaffold.yaml` and intercepts our Kubernetes-style `service.yaml` deployment manifest.
2.  **Dynamic Value Substitution:** Skaffold locates the abstract image placeholder (`image: bookshelf-app-image`) inside `service.yaml` and replaces it dynamically with the absolute, immutable image tag generated by Cloud Build during the CI phase.
3.  **Target Delivery:** Cloud Deploy consults the `Delivery Pipeline` defined in our Terraform code. It maps out the path, verifies the target environment parameters (currently pointing to the `bookshelf-prod` target), and verifies region availability (`europe-west1`).
4.  **Declarative Rollout:** The finalized, fully-rendered manifest is pushed directly to the Cloud Run API. Google Cloud processes the file and updates the live container seamlessly with zero downtime.
